# 22 - Resumen de implementacion (alineado)

## Stack base

```txt
Astro + Hono + Postgres
```

## Producto V1

```txt
Directorio SEO + proveedores + reviews moderadas + leads compuestos + panel proveedor + admin minimo.
```

## Decision critica

No implementar pagos internos de servicios ni garantia en V1.

## Micromercado inicial recomendado

```txt
Ciudad: Bogota
Vertical: servicios graficos/empaques
```

Servicios semilla:

- cajas-personalizadas,
- troqueles,
- impresion,
- serigrafia-screen,
- etiquetas,
- diseno-empaque,
- fotografia-producto,
- landing-page,
- ecommerce.

## Ruta de ejecucion recomendada

1. Fase -1: alineacion contractual y cierre de desalineaciones criticas.
2. Fase 0: plataforma base y calidad minima.
3. Fase 1: dominio y datos canonicos.
4. Fase 2: directorio publico SEO util.
5. Fase 3: leads compuestos end-to-end.
6. Fase 4: panel proveedor minimo.
7. Fase 5: admin minimo y moderacion.
8. Fase 6: wallet/cobro por oportunidad (V1.5).
9. Fase 7: cierre y reputacion.
10. Fase 8-9: hardening y go-live controlado.

## Primera validacion de mercado

Objetivos de viabilidad (antes de expansion):

- 10-20 proveedores activos,
- 25-60 leads/mes,
- 5+ reviews reales moderadas,
- 40%+ leads contactados,
- proveedores dispuestos a pagar por oportunidad valida.

## Regla de expansion

No abrir otra ciudad ni otro vertical hasta cumplir umbrales de viabilidad en el primer micromercado.
